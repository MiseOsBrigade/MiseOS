# Security policy

Do not open public issues for credential exposure, authorization bypass, unsafe auto-merge behavior, or remote-code-execution findings. Use GitHub private vulnerability reporting after it is enabled.

Never include live credentials.

Security commitments:
- no secret values in logs or reports;
- no automatic history rewrite;
- no production deployment without approval;
- least-privilege repository credentials;
- auditable policy decisions;
- model output cannot override fixed security policy.
