# Contributing

1. Open or select an issue.
2. Create a scoped branch.
3. Add or update tests.
4. Run `pnpm format:check`, `pnpm lint`, `pnpm typecheck`, `pnpm test`, and `pnpm build`.
5. Use Conventional Commits.
6. Document security and permission impact.

Changes to default permissions, approval rules, credentials, security boundaries, or auto-merge behavior require maintainer review.
